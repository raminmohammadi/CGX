# CGX optional ML extras.
#
# Install with:    pip install -r requirements.txt -r requirements-ml.txt
# Or via extras:   pip install -e .[embeddings]
#
# What this gives you:
#   * Local loading of the default Jina embedding model
#     (`jinaai/jina-embeddings-v2-base-code`) for indexing.
#   * The optional cross-encoder reranker
#     (`cross-encoder/ms-marco-MiniLM-L-6-v2`) used when
#     `HybridConfig.enable_reranker=True`.
#
# What you can skip this for:
#   * Using a BYO embedder (any callable that returns vectors).
#   * Running CGX purely against an Ollama or remote LLM endpoint.
#
# Note: `torch` is ~700 MB; pick a CUDA/CPU build that matches your platform.
# See https://pytorch.org/get-started/locally/ if the pin below doesn't apply.
#
# NVIDIA users: the wheel must match your driver, not just "be CUDA".
# `pip install torch` on PyPI defaults to the newest CUDA series, which can
# be ahead of your driver and silently falls back to CPU at runtime (only
# visible via `torch.cuda.is_available()` returning False and a slow-feeling
# embed/index pass). Pick the cuXXX channel that matches the "CUDA Version"
# column shown by `nvidia-smi`, e.g.:
#
#   # Driver supports CUDA 12.8 (most 5xx series drivers today):
#   pip install --index-url https://download.pytorch.org/whl/cu128 torch
#
#   # Driver supports CUDA 12.4 / 12.1 (older 5xx / 4xx series):
#   pip install --index-url https://download.pytorch.org/whl/cu124 torch
#   pip install --index-url https://download.pytorch.org/whl/cu121 torch
#
# CPU-only (any platform, no GPU): the default `pip install torch` is fine.
